Corporate UI Deployment Workflow

The development workflow follows a pretty straightforward process:

  1. Clone the UI repo from https://github.scania.com/Scania/corporate-ui.git
  2. Make your changes or add your new components to the repo, using the Corporate coding standard and guidelines.
  3. Test your changes according to the Corporate UI recomendations.
  4. Publish your branches.
  5. Make the necessary pull requests. Once approved, they will go into the next Corporate UI build.
